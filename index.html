API Reference

The Blockfreight API is organized around GraphQL. GraphQL is a query language for your API, and a server-side runtime for executing queries by using a type system you define for your data. We support cross-origin resource sharing, allowing you to interact securely with our API from a client-side web application (though you should never expose your secret API key in any public website's client-side code). JSON is returned by all API responses, including errors, which are indicated by HTTP response codes.

Once the Blockfreight node is running the api will be running on http://localhost:8080/bftx-api. As a GraphQL API, it does not have a route for each action, request body is where the action (or mutation) must be declared.

Errors

Blockfreight uses conventional HTTP response codes to indicate the success or failure of an API request. In general: Codes in the 2xx range indicate success. Codes in the 4xx range indicate an error that failed given the information provided (e.g., a required parameter was omitted, a charge failed, etc.). Codes in the 5xx range indicate an error with Blockfreight's servers.

HTTP status code summary
200 - ok Everything worked as expected.
400 - bad request The request was unacceptable, often due to missing a required parameter. Check whether the request body is correct.
406 - not acceptable The transaction has not been signed before trying to Broadcast or the transaction has been broadcasted and it is not possible to sign it again.
404 - not found Transaction not found.
500 - server errors Something went wrong on Blockfreight's end.

Core resources

See below the core Blockfreight resources, you will use them to create, sign, broadcast and/or query a transaction.

Remember that the base url is: http://localhost:8080/bftx-api. You will use it for all the resources.

Create a transaction.

Route: http://localhost:8080/bftx-api
Method: POST
Request Structure:
{
  "query": "mutation{constructBFTX(#BLOCKFREIGHT_TRANSACTION_ATTRIBUTES){ #BLOCKFREIGHT_TRANSACTION_ATTRIBUTES_THAT_MUST_BE_RETURNED }}"
}
Description:

This resource is responsible for creating a local transaction with the info sent inside the constructBFTX() and the return depends on the structure that is passed in the request.

package main

import (
    "fmt"
    "io/ioutil"
    "log"
    "net/http"
    "os"
)

func main() {
    client := &http.Client{}

    req, err := http.NewRequest("GET", "http://localhost:8080/bftx-api", nil)
    if err != nil {
        log.Print(err)
        os.Exit(1)
    }

    q := req.URL.Query()
    q.Add("query", `{tx: queryTransaction(id:"#BLOCKFREIGHT_TRANSACTION_ID"){Id Properties{DescOfGoods}}}`)
    req.URL.RawQuery = q.Encode()

    res, err := client.Do(req)

    if err != nil {
        log.Fatal(err)
    }
    bftxResult, err := ioutil.ReadAll(res.Body)
    res.Body.Close()
    if err != nil {
        log.Fatal(err)
    }
    fmt.Printf("%s", bftxResult)
}
